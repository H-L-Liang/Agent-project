1、为什么用 ReAct Agent 而不是普通 LLM 直接回答？
普通 LLM 只能靠训练数据回答，无法获取实时信息（天气、用户数据）也无法检索私有知识库。ReAct 让模型先思考需要什么信息，再调用对应工具获取，最后综合回答，解决了这两个问题。
2、RAG 是怎么工作的？
用户提问 → embedding 模型将问题向量化 → 在 ChromaDB 里做相似度检索 → 取出最相关的 k 条文档片段 → 拼入 prompt 让模型参考回答。核心是"检索增强"，让模型回答有据可查。
3、动态提示词切换怎么实现的？
用户触发"生成报告"意图时，Agent 会调用 fill_context_for_report 工具，这个工具本身不做事，但调用它会触发 LangGraph 中间件 report_prompt_switch，中间件检测到标记后将系统提示词替换为报告写手版本，后续模型输出风格就变了。
4、为什么要 MD5 去重？
知识库文档可能多次更新，每次重新加载如果不去重，同一内容会被重复向量化存入 ChromaDB，导致检索结果重复、数据库膨胀。用文件 MD5 做唯一标识，处理过的跳过，只处理新文件
5、工厂模式用在哪里？为什么？
model/factory.py 用工厂模式封装模型实例化，调用方只需要 chat_model 和 embedding_model 两个对象，不关心底层是通义千问还是其他模型。后续换模型只改工厂类，其他代码零改动。


能客观说的成果：
独立完成全链路开发：知识库构建 → RAG 检索 → Agent 推理 → 流式前端
实现两个完整业务场景：知识问答 + 个性化报告生成
代码具备工程化结构：配置驱动、日志、工厂模式、模块解耦
